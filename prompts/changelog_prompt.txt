Your task is to generate a professional CHANGELOG.md from git commit history.

INSTRUCTIONS:
- Group commits by type: Features, Bug Fixes, Improvements, Documentation, Chores
- Use conventional commit prefixes to classify: feat=Feature, fix=Bug Fix, docs=Documentation, refactor/perf=Improvement, chore/ci/test=Chore
- Write human-readable descriptions — expand abbreviations and terse messages
- Ignore merge commits, version bump commits, and WIP commits
- Group commits into logical releases if there are clear version tags; otherwise group by week

### 🐛 Bug Fixes
- Description of what was fixed

### ⚡ Improvements
- Description of performance or code quality improvements

### 📚 Documentation
- Documentation changes

### 🔧 Chores
- Build, CI, dependency updates
